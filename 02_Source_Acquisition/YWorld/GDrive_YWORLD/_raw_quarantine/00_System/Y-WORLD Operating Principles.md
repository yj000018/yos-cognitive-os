# Y-WORLD Operating Principles

> Y-WORLD is a World Operating System: a live cognitive infrastructure where my whole universe becomes navigable, structured, visual, semantic, agentic, and progressively executable.

## 1. Core Principles

* **Obsidian as Cognitive Interface**: Obsidian is the Human Cognitive Interface (HCI) to Y-WORLD. It is not just a notes app; it is a live cockpit.
* **Y-OS as Cognitive Subsystem**: Y-OS is the cognitive operating layer inside Y-WORLD, managing models, agents, automation, memory, routing, workflows, and execution.
* **Spatial Semantic Control**: Maps are spatial semantic control surfaces. The map is not decorative; it is an active control cockpit.
* **Durable Markdown Substrate**: The system must remain modular, readable, versioned, and portable markdown-first infrastructure.
* **Agentic Execution**: Manus is the agentic operator.

## 2. Structural Guidelines

1. **No Over-Engineering**: Keep the foundation clean, scalable, and modular.
2. **Strict Folder Organization**: Every file belongs to a defined region.
3. **Consistent Metadata**: Every note must have standard properties for queryability and automation.
4. **Plugin Hygiene**: Do not activate everything by default. Separate installed, active, and visible commands.
5. **Durable Portability**: Avoid fragile automations that break when offline or without specific plugins.
