## 3. Core doctrines decided

### 3.1 Architecture before extraction

Decision:

**The current priority is to build and validate the machine, not to run full automatic extraction.**

Preserved wording:

> On construit la machine avant de lui donner toute la nourriture.

Meaning:

1. Validate structures.
2. Validate protocols.
3. Validate registries.
4. Validate gates.
5. Validate roles.
6. Validate connector and pipeline designs.
7. Only later decide minimal targeted acquisitions.

Do not interpret this as a permanent ban on extraction. It is a current-phase control doctrine.

---

### 3.2 Git/Markdown is the only knowledge repository

Decision:

**Git + Markdown is the source of truth for knowledge.**

Implications:

1. Durable knowledge belongs in Git.
2. Markdown is the primary human-readable format.
3. JSON is allowed for registries, manifests, schemas, and machine-readable control files.
4. Obsidian reads from Git/Markdown.
5. Notion is not a repository.
6. Mem0 is not a repository.

---

### 3.3 Obsidian is consultation/navigation only

Decision:

**Obsidian is the consultation and navigation layer over Git/Markdown.**

Implications:

1. Obsidian should not become a separate source of truth.
2. Avoid Obsidian-only proprietary structures as canonical data.
3. The Git repo should be structured so Obsidian can navigate it.
4. Future work should include README/index files, backlinks, project pages, protocols, registries, and resource navigation.

---

### 3.4 Notion is frozen legacy

Decision:

**Notion is no longer an active knowledge repository.**

Notion status:

1. Frozen / legacy source.
2. Future migration target.
3. Future decommission target.
4. Must not receive new knowledge as repository.
5. Useful Notion knowledge should be reacquired and moved to Git/Markdown.
6. Future sprint likely: `WP2-NOTION-DECOMMISSION`.

Important nuance:

This does not mean Notion data is useless. It means Notion is a **legacy source to migrate**, not a continuing backend.
