# Canonical Doctrine Registry

> Y-OS Control Plane — Active Doctrines
> Last updated: 2026-07-03

## Purpose

This registry contains all active canonical doctrines governing Y-OS and KAP. Doctrines here are the durable authority. They supersede any version found only in ChatGPT conversation history or Manus session context.

---

## Active Doctrines

### DOCTRINE-YOS-001 — YOS Contains KAP

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

Y-OS is the global cognitive operating system. KAP is a module/process inside Y-OS. KAP handles knowledge assimilation, source acquisition, merge architecture, and synthesis workflows. Y-OS includes KAP but also: memory, agents, pipelines, routing matrices, self-improvement loops, coordinated agent teams, automation, capability expansion, cognitive RAM, context systems, model routing, agent routing, human/AI exploitation layers, CasaTAO, and all long-term cognitive infrastructure.

---

### DOCTRINE-YOS-002 — Architecture Cognitive Avant Absorption Massive

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

KAP/YOS must finalize and validate cognitive architecture before broad source absorption. No broad acquisition, normalization, synthesis, or merge should happen before defining:
- Source fragment model
- Claim model
- Thought-line model
- Decision-thread model
- Contradiction and supersession policy
- Impasse registry
- Current-best-knowledge protocol
- Human/AI exploitation model
- Gate roadmap

---

### DOCTRINE-YOS-003 — Git is the Durable Authority

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

New strategic doctrine must not remain only in ChatGPT. ChatGPT may generate doctrine. Manus must persist doctrine. Git is the durable authority. All durable outputs must be written as Markdown/JSON files inside a Git-controlled structure.

---

### DOCTRINE-YOS-004 — Control Plane vs Source Staging

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

New active doctrines and decisions go into the Y-OS Control Plane (`00_Control_Plane/`, `01_Strategy/`, `02_Architecture/`). Recovered historical sources from Notion, Manus, ChatGPT, Obsidian, Git, Markdown, and Web go into Source Staging (`06_Source_Staging/`) until reviewed. Source Staging is not canonical truth. Future merge will link active doctrines to supporting, contradicting, or superseded historical sources.

---

### DOCTRINE-YOS-005 — No Current-Best Synthesis Before Model Validation

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

Current-best syntheses are not authorized until the Source Fragment, Claim, Thought Line, Decision Thread, Contradiction/Supersession, and Human/AI Review models are validated.

---

### DOCTRINE-YOS-006 — Manus is High-Value but Selective

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

Manus is a very high-value execution corpus containing MPMs, gate reports, durable outputs, architecture decisions, execution traces, and program states. However, Manus must not be blindly imported as a flat task corpus. Use metadata census, relevance scoring, durable output detection, noise filtering, and selective deep extraction only after gate authorization.

---

### DOCTRINE-YOS-007 — Obsidian is Easy but Gated

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

Obsidian/Local Markdown is a high-feasibility pipeline because its source format is close to the Y-OS/KAP target format: Markdown, folders, wikilinks, frontmatter, and local assets. However, real vault scan, import, normalization, link rewriting, and source mutation remain forbidden until explicit gate authorization.

---

### DOCTRINE-YOS-008 — Notion Active Controlled Pipeline

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: ChatGPT KAP/YOS session

Notion access has been granted to Manus and cleanup has been reported as completed. Notion may operate as an active controlled pipeline, but uncontrolled full workspace dump, canonical merge, broad acquisition, and source mutation remain forbidden unless explicitly authorized by future gates.

---

### DOCTRINE-YOS-009 — One Engine, Multiple Scopes

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: Manus KAP session

KAP is a single engine that processes all sources. Scope is a tag on fragments (`scope:yos`, `scope:youniverse`, `scope:both`), not a pipeline branch. The same acquisition → fragment → classify → route → synthesize logic applies to all sources. YOUniverse sources (GDrive, Calendar, Email, YouTube, etc.) are catalogued now and will be activated in Phase 2 — they are not permanently excluded.

---

### DOCTRINE-YOS-010 — Source Registry System Modularity

**Status**: ACTIVE | **Date**: 2026-07-03 | **Source**: Manus KAP session

Every source in KAP follows a lifecycle: Discovered → Catalogued → Connector Designed → Configured → Active → Paused → Deprecated. No source injects data directly into the synthesis layer — it must pass through a Connector and become a Source Fragment first. Every fragment carries its Source ID, Connector Version, and Scope Tags.
