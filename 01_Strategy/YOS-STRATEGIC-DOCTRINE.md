# Y-OS Strategic Doctrine

> Last updated: 2026-07-03

## Vision

Y-OS is a cognitive operating system for an individual — a living, evolving infrastructure that captures, synthesizes, routes, and exploits all knowledge produced by and about Yannick Jolliet. It is not a tool. It is an architecture.

## Strategic Principles

**1. Architecture before absorption.** Define the cognitive model before ingesting data. A well-defined model prevents garbage-in-garbage-out at scale.

**2. Git is the durable authority.** Any doctrine, decision, or architecture that exists only in a conversation is at risk. Persistence to Git is mandatory.

**3. One engine, multiple scopes.** The same KAP pipeline processes all sources. Scope tags (`scope:yos`, `scope:youniverse`, `scope:both`) route fragments to the correct destination. No pipeline forking.

**4. Selective, not exhaustive.** Not all sources are equal. Manus sessions are high-value but selective. Obsidian is easy but gated. Notion is active but controlled. ChatGPT is core but requires export handling.

**5. Evolutionary merge, not overwrite.** Knowledge evolves. Contradictions and supersessions are first-class citizens. No fragment is silently overwritten — it is superseded with a traceable link.

**6. Human/AI exploitation model.** Y-OS is designed to maximize the cognitive leverage of a single human (Yannick) through coordinated AI agents. The system serves the human, not the reverse.
