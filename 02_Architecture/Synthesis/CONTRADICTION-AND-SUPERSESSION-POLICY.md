# Contradiction and Supersession Policy

## 1. Core Principle
Knowledge evolves. Contradictions are not errors; they are evidence of evolution, context shifts, or unresolved debates. The system must preserve contradictions until they are explicitly superseded or resolved.

## 2. Handling Contradictions
When a new claim contradicts an existing claim in the Current Best Synthesis:
1. **Quarantine:** The new claim is held in a "Contested" state.
2. **Context Evaluation:** The system evaluates the context (date, source authority, domain).
3. **Thought Line Update:** Both claims are linked within the relevant Thought Line, explicitly marking the contradiction.

## 3. Supersession Rules
A claim or decision is superseded ONLY when:
- **Explicit Override:** A newer source explicitly states it replaces the older one.
- **Authority Hierarchy:** A higher-authority source (e.g., Guardian Architect directive) contradicts a lower-authority source (e.g., a draft note).
- **Temporal Weight (with caution):** A significantly newer implementation consistently contradicts an older design doc.

## 4. Resolution Protocol
- If supersession is clear, the older claim is marked `SUPERSEDED` and the new claim becomes `ACTIVE`.
- If ambiguous, an **Impasse** is generated and flagged for Guardian review. The Current Best Synthesis reflects the ambiguity (e.g., "Status: Contested between Option A and Option B").
